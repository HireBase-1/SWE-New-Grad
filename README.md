## 🔦 Weekly Spotlight

### “New‑Grad Roles Hiding in Plain Sight”

If you’ve only looked at the big‑name career pages, it’s easy to think new‑grad SWE roles are either:

1. A hyper‑structured program at a FAANG‑ish giant, or  
2. Random “junior” postings that secretly want 3+ years of experience.

Once you slice the market down to **0–2 YOE, SWE‑ish titles, last 14 days**, a different pattern shows up:

- A lot of opportunities are at **mid‑size product companies** (Fintech, SaaS, infra tools) that don’t loudly brand their roles as “New Grad” but clearly expect early‑career engineers.
- You see **rotation programs** (like D2L) that let you try multiple teams before you specialize.
- There’s a surprising number of **data‑ and infra‑adjacent roles** (CERN, Morningstar, BBC) that are totally fine with junior engineers as long as you’re comfortable with Python, Linux, and some basic cloud.

If you’re still figuring out your path, it’s perfectly reasonable to hedge:

- One application push to **rotation / associate programs** where the learning scaffolding is explicit.
- One push to **smaller product teams** where you’ll touch a lot of the stack and get reps shipping.
- One push to a **data / ML‑adjacent team** where Python + SQL + a bit of modeling literacy go a long way.

You don’t have to pick your forever stack at 22. You just need a place where you’ll actually write code, ship it, and learn from the fallout.
