## 🔦 Weekly Spotlight

### “AI‑flavored, but still very much Software Engineering”

This week’s slice of the market has a fun pattern: a lot of the best new‑grad‑friendly roles are **AI‑adjacent without being pure research**.

You see:

- Classic product companies (Airbnb, SmithRx, Blink) hiring early‑career engineers into stacks that quietly rely on **recommendation systems, experimentation platforms, and analytics**.
- Dedicated AI teams (TWG Global AI, Abnormal Security, Affirm) advertising roles that still look like normal backend engineering, just with more **Python, data plumbing, and model integration** in the mix.

If you’re coming out of school right now, that’s actually good news:

- You don’t need a PhD or a “Research Scientist” title to be close to AI work.
- The thing that still matters most is whether you can **ship reliable software**: clean APIs, decent tests, and enough observability that you can debug the weird production bugs.

A practical way to position yourself:

- One project that looks like a **normal product** (REST/GraphQL API, database, simple front‑end).
- One project that is **AI‑adjacent**, but scoped: maybe retrieval‑augmented search over a small dataset, or an internal tool that uses an LLM to summarize logs.

That combo maps surprisingly well onto the real roles that keep showing up in this feed.
